## Pre-approved triage policy

### AUTO-APPLY (no asking; just verify import + commit)
- **Formula-exact dup pairs** -- delete higher-numbered duplicate (def + registry entry).
  Keep lower-numbered as canonical. AST-parse-check the file. Commit.

### VERIFY NUMERICALLY then APPLY (3 signals, <1e-9 tolerance, no asking if all pass)
Standard patterns (already proven this session):
- `rolling(W).apply(lambda x: pd.Series(x).autocorr(lag=1), raw=False)`
  -> `rolling(W-1).corr(s.shift(1))`
- `for b, val in zip(digitized, vals): profile[b] += val`
  -> `np.bincount(digitized, weights=vals, minlength=n_bins)`
- Per-i recomputation of `s.iloc[idx-N:idx+1].pct_change().dropna()`
  -> pre-compute `ret = s.pct_change()` once, slice into `.values`
- Inner Python list-comp `np.array([x[i:i+L] for i in range(n-L)])`
  -> `sliding_window_view(x, L)[:n-L]` + broadcast ops
- O(N^2) inner double-for over (i, j) pairs with chebyshev distance
  -> 4-D broadcast + `np.triu` mask + reduction

If verification fails on any of the 3 signals -> escalate (see below).

### ASK FIRST (do not auto-apply)
- Value-exact dups that aren't formula-exact (might be intentional dual-naming OR hidden bug)
- Constants at entry day (could be design choice OR implementation flaw -- e.g. `close.abs()` no-op for equities)
- Naming corrections (e.g. function named `momentum_quality` that actually computes Calmar)
- Any rename or restructure beyond delete-and-rebind
- Vectorizations whose net speedup is unclear (memory-bound regimes)
- New error class not previously documented

### STOP and ESCALATE
- Harness fails to import a module (NameError on registry entry, etc.)
- AST parse fails after edit
- Runtime > 1 hour for a single family even after applying audit fixes
- New unbindable column type not yet inventoried
- Verification fails at 1e-9 tolerance on any signal

## Verification recipe (canonical)

```python
import numpy as np, pandas as pd, importlib.util, time

# Load OLD from source; never hand-roll the "old" version.
spec = importlib.util.spec_from_file_location("mod", "<path>")
mod = importlib.util.module_from_spec(spec); spec.loader.exec_module(mod)
old_fn = mod.<function_name>

# Define new_fn inline.

# 3-signal test (1500 elements each).
np.random.seed(42)
SIGNALS = {
    'rand_walk': pd.Series(100 + np.random.randn(1500).cumsum() * 0.5),
    'flat':      pd.Series(np.full(1500, 100.0)),
    'sinusoid':  pd.Series(100 + 5 * np.sin(np.linspace(0, 50*np.pi, 1500))),
}
for name, sig in SIGNALS.items():
    old, new = old_fn(sig), new_fn(sig)
    assert np.allclose(old.values, new.values, rtol=0, atol=1e-9, equal_nan=True), name
```
